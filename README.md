# HR AI Assistant
_Проект носит демонстрационный характер. Преследовалась цель создания устойчиво работающего MVP.
Структура папок проекта, маниакальная обработка ошибок и исключений, тесты и прочие best-practices временно отложены._

Проект представляет собой систему, которая автоматически проверяет директорию с документами, превращает их в контекст LLM и использует для ответов на вопросы пользователя, как локальную базу знаний.

## Описание работы

Система состоит из двух параллельно работающих компонентов, организованных через треды:

**Фоновый сборщик** автоматически парсит папку с документами с заданной периодичностью.

Используется дедупликация — если документ был добавлен ранее, он не будет обработан повторно. 

Текст каждого документа разбивается на чанки через LangChain Text Splitters. Затем чанки преобразуются в векторные представления с помощью модели эмбеддингов и сохраняются в векторную базу данных Chroma.

**Интерфейс вопрос-ответ** работает в основном потоке и ожидает ввода пользователя в консоли. 

После получения запроса от пользователя, система ищет в векторной базе наиболее релевантные чанки, формирует контекст и отправляет его вместе с запросом в локальную LLM. 

Модель генерирует ответ исключительно на основе найденной информации, строго следуя заданному формату, без обращения к внешним API.

## Стек

- **Python 3.14** — основная среда разработки
- **LangChain** — фреймворк для работы с текстовыми сплиттерами и построения RAG
- **Ollama** — локальный сервер для запуска языковых моделей
- **ChromaDB** — векторная база данных для хранения и поиска семантических соответствий
- **Requests** — загрузка данных с веб-сайтов и RSS-лент

## Возможности

- Автоматический сбор документов в отдельном потоке 
- Дедупликация документов
- Разбиение на чанки с перекрытием
- Локальное векторное хранилище
- Консольный интерфейс для вопросов с потоковой генерацией ответов
- Полная приватность — все данные и модель работают локально
