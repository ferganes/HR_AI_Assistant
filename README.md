## HR AI Assistant — Local Document Intelligence System

Автономный AI-ассистент для HR-департамента, работающий полностью локально. Система мониторит директорию с документами в реальном времени, автоматически индексирует новые файлы и отвечает на вопросы сотрудников через RAG-архитектуру с семантическим поиском. Все данные остаются внутри компании — zero leakage, zero cloud costs.

### Архитектура и ключевые решения
- Параллельный пайплайн обработки. Фоновый поток непрерывно сканирует директорию, дедуплицирует документы по хешу контента и накапливает векторное хранилище. 
- Основной поток обрабатывает пользовательские запросы без блокировок — конкурентная обработка без race conditions.
- Продвинутая RAG-оркестрация. Кастомная стратегия чанкинга с перекрытием через LangChain Text Splitters сохраняет межчанковый контекст. 
- Ретривал через ChromaDB, ответы строго ограничены индексированным контентом — исключены галлюцинации модели.
- Оптимизация под локальное железо. 3B-модель Qwen 2.5 через Ollama на CPU с приемлемой latency. BGE-M3 для мультиязычных эмбеддингов.
- Полная автономность: работает без интернета, без API-ключей, без подписок.

### Результат
Превращает неструктурированный документооборот HR в структурированную базу знаний с мгновенным семантическим доступом. End-to-end pipeline: ingestion → chunking → embedding → retrieval → generation — полностью offline, приватно, масштабируемо.

### Стек

- **Python 3.10** — основная среда разработки
- **LangChain** — фреймворк для работы с текстовыми сплиттерами и построения RAG
- **Ollama** — локальный сервер для запуска языковых моделей
- **ChromaDB** — векторная база данных для хранения и поиска семантических соответствий
